# Небольшой стенд для демонстрации работы модели для обнаружения XSS атак

## Структура файлов
- `xssdetector` - файл со всеми данными для обучения модели и её запуска как микросервиса;
- `server` - основной микросервис, который выполняет роль главной backend-части приложения;
- `ssrw-client` - мини-клиент, изображающий возможную ситуацию эксплуатации уязвимости;
- `остальные файлы` - всё остальное в общем и целом не столь значимо, кроме файлов для docker-а.

## Принцип работы

1. Клиент отправляет запрос к серверу;
2. Сервер получив request отправляет текст сообщения в модель на анализ;
3. Анализ ответа модели:
    a. Если модель ответила, что это не потенциально опасный код, то сервер записывает сообщение в БД и отправляет ответ 204;
    b. Если модель ответила, что это XSS, то сервер не записывает сообщение в БД и отправляет ошибку 400 с текстом `Обнаружен подозрительный код`
4. Клиент получает ответ от сервера и либо отображает все комментарии, либо отображает ошибку
